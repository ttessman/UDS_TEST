# User Journeys

## Evaluator

An evaluator opens the catalog and sees live package state instead of a static list. The page shows UDS readiness, installed Package CRs, Store entries, and backend command output for trust and debugging.

## Normal User

A normal user finds an installed resource card, opens its launch menu, and starts the app through a named UDS endpoint such as `docs.uds.dev` or `app.uds.dev`.

## Admin User

An admin reviews available Store entries, installs an approved package, watches backend command feedback, and sees the installed resource appear after the cluster reports it.
